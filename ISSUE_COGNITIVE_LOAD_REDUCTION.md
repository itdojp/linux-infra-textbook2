# ISSUE: Linux Infra Textbookの認知負荷軽減と理解度向上

## 📊 現状分析

### 3冊の書籍の現在の位置づけ
1. **illustrated-linux-basics-book**（初級）
   - 必須コマンド15選、図解中心
   - エラー対処重視
   - 5章構成、非常にコンパクト

2. **wsl2-linux-essentials-book**（中級）
   - WSL2での実践
   - viエディタ、systemd、WordPress構築
   - 6章構成、実践的

3. **linux-infra-textbook**（上級）
   - 理論的背景から深く理解
   - Podman、AWS、IaCまでカバー
   - 15章構成、包括的

### 問題点
- **難易度の急激な上昇**: 中級から上級への飛躍が大きすぎる
- **抽象度が高い**: 哲学的・理論的な説明が多く、初学者には理解困難
- **実践との距離**: 理論先行で手を動かす機会が後半に偏る
- **専門用語の壁**: 説明なしに高度な用語が登場

## 🎯 改善提案

- 第1章から「ITインフラの全体像」という抽象的な話

- 実践は第4章から

### 2. 認知負荷を下げる工夫

#### A. プログレッシブ・ディスクロージャー
各概念を3段階で説明：
```
Level 1（概要）: 1行説明「〜とは〜するもの」
Level 2（具体例）: 身近な例での説明
Level 3（技術詳細）: 正確な技術的説明
```

例：
```markdown
## コンテナとは

**Level 1**: アプリケーションを箱に入れて持ち運べるようにする技術

**Level 2**: 引っ越しのダンボール箱のようなもの
- 必要なものを全部入れる（アプリ＋ライブラリ）
- どこでも開けばすぐ使える（環境依存なし）
- 他の箱と混ざらない（隔離）

**Level 3**: Linuxカーネルの名前空間とcgroupsを使った...
```

#### B. ビジュアル要素の強化
- [ ] 各章冒頭に「章の地図」（全体における位置づけ）
- [ ] 複雑な概念には必ず図解を追加
- [ ] コマンド実行結果のスクリーンショット
- [ ] Before/Afterの比較図

#### C. スモールステップ学習
- [ ] 各セクションを10分で読める分量に
- [ ] セクション末に「理解度チェック」（Yes/No質問）
- [ ] 章末に「できるようになったこと」チェックリスト

### 3. 実践機会の増加

#### 現状の問題
- 演習問題が章末にまとめて配置
- 理論を全部読んでから実践

#### 改善案
```markdown
## インラインハンズオン
理論説明の直後に「今すぐ試そう」ボックス

### 例：プロセスの説明後
📝 **今すぐ試そう（30秒）**
```bash
# 現在のプロセスを見てみよう
ps aux | head -5

# あなたのプロセスだけ表示
ps aux | grep $USER
```
何個のプロセスが動いていましたか？
```

### 4. 前提知識の明確化と補完

#### 各章の冒頭に追加
```markdown
## 📚 この章の前提知識
- ✅ 必要：基本的なLinuxコマンド（ls, cd, cat）
- ✅ 必要：IPアドレスの概念
- ❌ 不要：プログラミング経験
- ❌ 不要：ネットワーク詳細知識

足りない知識がある場合 → [第X章を先に読む] or [クイック補講（5分）]
```

### 5. エラーベースラーニング

#### 各技術説明に「よくある間違い」を追加
```markdown
## ⚠️ よくある間違いと対処法

### Podmanでイメージが見つからない
```bash
# 間違い
podman pull ubuntu  
# エラー: Error: short-name "ubuntu" did not resolve to an alias

# 正解
podman pull docker.io/ubuntu
# Podmanはフルパスが必要
```

### なぜこのエラーが出る？
DockerはDocker Hubがデフォルト、Podmanは...
```

### 6. 用語集とグロッサリー

#### 各章末に追加
```markdown
## 📖 この章の重要用語
| 用語 | 読み方 | 1行説明 | 詳細 |
|-----|--------|---------|------|
| cgroups | シーグループス | リソース制限の仕組み | p.123 |
| namespace | ネームスペース | 隔離の仕組み | p.125 |
```

### 7. 段階的な難易度設定

#### 章内での難易度表示
```markdown
### 🟢 基礎：ファイルの権限（全員必須）
### 🟡 応用：ACLの使い方（余裕があれば）
### 🔴 発展：SELinuxの設定（上級者向け）
```
